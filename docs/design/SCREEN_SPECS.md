# 页面与交互规格

配套总览图：[UI 设计总览](wireframes/design-board.svg)。这是结构与视觉方向稿；正式美术资源、牌面和插画需要单独完成原创与授权审查。

## 1. 启动页

- 品牌标识、加载进度、当前动作（校验版本/连接服务/加载大厅）。
- 8 秒无进展显示重试与诊断编号；弱网不能只显示旋转图标。
- 静默恢复上次未结束牌局，检测到有效 session 时优先展示“返回牌局”。

## 2. 大厅

- 顶栏：头像昵称、连接状态、公告、设置。
- 主区：继续对局（如有）、创建好友房、加入房间；目录先按“纸牌类 / 麻将类 / 聚会推理”分组，再展示具体玩法。
- 首屏只突出“创建房间”；加入房间是明确的次级入口。
- 分类需要容纳地区或模式扩展：纸牌类包含斗地主、炸金花、德州等；麻将类包含四川麻将、贵州麻将、国标麻将等。
- 具体游戏卡必须展示人数、预计时长和“可玩/规划中/规则研究”状态，避免用户进房后才发现不合适。

验收：横屏 16:9 不滚动即可完成分类切换、创建/加入；断网时创建和加入入口禁用并说明；规划中卡片只显示规划信息，不创建房间。

## 3. 创建房间

- 左侧选择游戏，右侧显示该游戏独立规则表单。
- 规则按“常用/高级”分组，默认值可直接开局。
- 底部固定显示创建按钮；创建中防重复点击。
- 规则摘要进入候场页并在开局前可由房主修改，修改会取消其他玩家准备。

## 4. 候场房间

- 房间码和分享按钮最醒目；显示游戏、规则摘要和 3 个席位。
- 玩家可准备、换座、离开；房主可改规则、移交、踢出（需确认）和开始。
- 开始按钮仅当人数正确且全员准备时启用，并显示未满足原因。
- 断线玩家保留在原席位，头像显示离线与等待时长。

## 5. 斗地主牌桌

- 顶部两个对手，底部本地玩家；中间是上一次出牌与阶段提示。
- 自己的 17–20 张牌必须在常见手机宽度下可选；牌多时重叠，但点数与花色始终露出。
- 操作区根据合法动作显示“提示 / 不出 / 出牌”或叫抢按钮；非法按钮不出现。
- 选牌后展示牌型反馈；服务端确认前按钮进入 pending，失败则恢复选择。
- 倍数、剩余牌数、托管、网络状态在同一信息层，不遮挡牌面。

验收：所有信息来自观察者投影；截屏不可看到其他玩家手牌；前后台恢复后牌序、选中态和当前回合正确。

## 6. 结算

- 先展示胜负，再列本局倍数构成、三人得分、关键事件。
- 主操作“再来一局”，次操作“返回房间”；分享战绩不可包含敏感身份或完整牌局。
- 结算记录 ID 可复制，用于问题反馈和审计定位。

## 7. 通用状态

每个页面都要设计：加载、空数据、断网、权限拒绝、服务维护、版本过低、操作成功和可恢复错误。危险确认不可与普通提示共用同一种按钮颜色。
