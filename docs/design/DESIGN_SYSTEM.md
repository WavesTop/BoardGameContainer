# 视觉与交互设计系统 v0.1

## 设计方向

关键词：**深夜好友局、现代桌面、清晰可信、轻松但不幼稚**。大厅像一间安静的桌游俱乐部，牌桌强调材质和层次；不复刻任何商业桌游的商标、卡面或角色美术。

首版以横屏 `16:9` 设计，逻辑尺寸 `1600 × 900`。所有关键按钮和手牌都必须落在安全区，适配刘海、圆角和微信胶囊区域；窄屏允许压缩留白，不改变操作顺序。

## 颜色

| Token | 色值 | 用途 |
| --- | --- | --- |
| `bg/deep` | `#071A1B` | 页面最深背景 |
| `surface/green` | `#0D3331` | 大厅与牌桌主表面 |
| `surface/raised` | `#164844` | 卡片、浮层、侧栏 |
| `brand/gold` | `#F4C96B` | 主要强调、房间码、地主标记 |
| `action/coral` | `#F06B5D` | 主要行动、警示倒计时 |
| `state/mint` | `#64D6B4` | 在线、准备、成功 |
| `text/primary` | `#F7F3E8` | 主文字 |
| `text/secondary` | `#A9C4BD` | 次文字 |
| `line/subtle` | `#2C5A55` | 分隔线与描边 |

对比度目标：正文至少 WCAG AA；状态不能只用颜色表达，必须配图标或文字。

## 字体与层级

首选系统中文无衬线字体；数字和房间码使用等宽数字。小游戏正式包需确认字体授权并做子集化。

- Display：48/60，房间码与胜负标题。
- H1：32/42，页面标题。
- H2：24/32，卡片和模块标题。
- Body：20/30，正文与按钮。
- Caption：16/24，状态与辅助信息。
- 牌面数字：32–44，必须在低端机和小屏上可辨认。

## 间距与形状

使用 8px 网格：`8 / 16 / 24 / 32 / 48 / 64`。卡片圆角 20，按钮圆角 16，标签圆角 999。外层阴影克制，牌桌信息优先依靠明度和描边分层。

## 核心组件

- **Primary Button**：高 64，金色底深色字；一屏最多一个最高优先级按钮。
- **Secondary Button**：透明或深绿底，1px 描边。
- **Danger Button**：珊瑚红，仅用于退出、解散等不可逆动作。
- **Game Card**：封面、名称、人数、时长、状态和一个明确入口；未开放游戏展示“规划中”，不可点击。
- **Player Seat**：头像、昵称、在线、准备/托管、房主/身份标记；断线不移除席位。
- **Timer Ring**：数字 + 环形进度；5 秒内转珊瑚红并轻微脉冲，不用高频闪烁。
- **Toast**：最多两行，错误要说明可恢复动作。
- **Modal**：危险操作要求标题、后果、取消与确认，焦点默认在取消。

## 动效与声音

- 微交互 120–180ms，页面切换 220–320ms，发牌允许 500–900ms 的序列动画。
- 权威事件进入动画队列；动画可以加速/跳过，不能阻塞状态应用。
- 断线重连时暂停交互并展示“正在恢复牌局”，恢复成功后从快照重建桌面。
- 提供总音量、音效、音乐、震动开关；微信前后台切换必须正确暂停与恢复音频。

## 无障碍与易用性

- 触控目标至少 48×48 逻辑像素，关键行动建议 64 高。
- 不能依赖 hover；选牌同时使用抬升、描边和轻触反馈。
- 牌色除红黑外增加花色符号，色弱模式提高明度差。
- 所有等待状态给出“在等谁/还能做什么/多久后发生什么”。
